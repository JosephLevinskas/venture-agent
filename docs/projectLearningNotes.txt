# Local vs Docker Database URLs
- Local Mac Python connects to Postgres through mapped host port.
- Local DATABASE_URL = postgresql://postgres:postgres@127.0.0.1:5433/venture_agent
- Docker backend connects by Compose service name.
- Docker DATABASE_URL = postgresql://postgres:postgres@postgres:5432/venture_agent
- 127.0.0.1 inside a container means that same container, not your Mac.
- postgres:5432 works inside Docker Compose because postgres is the service name.

# SQLAlchemy Key Facts
- SQLAlchemy model = database table shape.
- Pydantic schema = API input/output shape.
- Base = parent class for models.
- engine = database connection manager.
- SessionLocal = creates database sessions.
- session = one temporary conversation with the database.
- get_db() opens session, yields it, then closes it.
- yield gives db to route and continues later for cleanup.
- db.add() stages a new row.
- db.commit() saves changes.
- db.refresh() reloads database-generated values like id.
- db.delete() deletes a row.
- .all() returns all matching rows.
- .first() returns first match or None.
- HTTPException returns API errors.
- exclude_unset=True prevents PATCH from overwriting missing fields with None.

# Password Rules
- Never store plain passwords.
- Store hashed_password only.
- Never return password or hashed_password from API.
- Register flow: plain password -> hash -> store hash.
- Login flow: submitted password + stored hash -> verify true/false.

# JWT Rules
- JWT = signed login token.
- SECRET_KEY signs tokens.
- Do not commit real SECRET_KEY.
- HS256 should use a long secret key, at least 32 bytes.
- access_token goes in Authorization header.
- Header format: Authorization: Bearer <token>.
- Token payload currently uses sub=email.
- exp means expiration time.
- decode_access_token verifies signature and expiration.
- Invalid/expired/fake token should fail.

# get_current_user
- Reads bearer token from Authorization header.
- Decodes JWT.
- Gets email from token subject.
- Loads user from database.
- Returns current user.
- Protected route uses current_user: models.User = Depends(get_current_user).

# Auth vs Authorization
- Authentication = who are you?
- Authorization = are you allowed to access this thing?
- JWT solves authentication.
- owner_id checks solve authorization.

# Alembic Commands
- .venv/bin/alembic current = show current DB migration.
- .venv/bin/alembic history = show migration history.
- .venv/bin/alembic revision --autogenerate -m "message" = create migration from model changes.

# Document Security Rules
- All document routes require auth.
- User can only create documents on projects they own.
- User can only list documents for projects they own.
- User can only get/update/delete documents attached to their own projects.
- For single document routes, use join Document -> Project to check owner.
- Return 404 for another user's document.

# Cascade Delete
- Project.documents uses cascade="all, delete-orphan".
- If a project is deleted, attached documents should be deleted too.
- This prevents orphan documents.

# Example Auth Curl Flow
- Register: curl -X POST "http://localhost:8000/auth/register" -H "Content-Type: application/json" -d '{"email":"joe@example.com","password":"password123"}'
- Login: curl -X POST "http://localhost:8000/auth/login" -H "Content-Type: application/json" -d '{"email":"joe@example.com","password":"password123"}'
- Save token: TOKEN="paste_access_token_here"
- Me: curl "http://localhost:8000/auth/me" -H "Authorization: Bearer $TOKEN"

# Example Project Curl Flow
- Create project: curl -X POST "http://localhost:8000/projects" -H "Content-Type: application/json" -H "Authorization: Bearer $TOKEN" -d '{"title":"My project","description":"notes"}'
- List projects: curl "http://localhost:8000/projects" -H "Authorization: Bearer $TOKEN"
- Update project: curl -X PATCH "http://localhost:8000/projects/1" -H "Content-Type: application/json" -H "Authorization: Bearer $TOKEN" -d '{"title":"Updated title"}'
- Delete project: curl -X DELETE "http://localhost:8000/projects/1" -H "Authorization: Bearer $TOKEN"

# Example Document Curl Flow
- Create document: curl -X POST "http://localhost:8000/projects/1/documents" -H "Content-Type: application/json" -H "Authorization: Bearer $TOKEN" -d '{"title":"Research Note","content":"These are my notes."}'
- List documents: curl "http://localhost:8000/projects/1/documents" -H "Authorization: Bearer $TOKEN"
- Get document: curl "http://localhost:8000/documents/1" -H "Authorization: Bearer $TOKEN"
- Update document: curl -X PATCH "http://localhost:8000/documents/1" -H "Content-Type: application/json" -H "Authorization: Bearer $TOKEN" -d '{"title":"Updated Note"}'
- Delete document: curl -X DELETE "http://localhost:8000/documents/1" -H "Authorization: Bearer $TOKEN"

# Common Errors
- DATABASE_URL is not set = .env missing or not loaded.
- connection refused = Postgres probably not running.
- password authentication failed = old Postgres volume credentials or wrong .env.
- column projects.owner_id does not exist = migrations not applied or old DB.
- relation documents does not exist = documents migration not applied.
- route not in /docs = router not included in main.py.
- ModuleNotFoundError for app = run with PYTHONPATH=backend.

# Fixes
- Start DB: docker compose up -d postgres.
- Apply migrations: .venv/bin/alembic upgrade head.
- Check current migration: .venv/bin/alembic current.
- Check tables: docker compose exec postgres psql -U postgres -d venture_agent then \dt.
- Run tests: PYTHONPATH=backend .venv/bin/pytest backend/tests -q.
- Fix import path: use PYTHONPATH=backend.
- If local dev DB is totally broken and data does not matter: docker compose down -v then docker compose up -d postgres then .venv/bin/alembic upgrade head.
